#### 三、调整网卡中断合并(interrupt coalescing)参数
```bash
ethtool -C eth20 rx-usecs 64 rx-frames 128
ethtool -C eth21 rx-usecs 64 rx-frames 128
```

#### 四、中断率的计算

1） **已知条件**

  通过sar命令观察对应的PPS:

  ```text
  每 NIC PPS: ~6M（总 12M / 2 张卡）
  每 NIC 队列数: 63
  每队列 PPS: 6,000,000 / 63 ≈ 95,000 pps
  每队列包间隔: 1,000,000 µs / 95,000 ≈ 10.5 µs
  ```

2）**rx-usecs=3 时**

  ```text
  第一个包到达 → 启动 3 µs 计时器
    ↓ 3 µs 后
  窗口内只有 ~0.3 个额外包 → 实际只处理 1 个包
    ↓
  中断频率 ≈ PPS ≈ 95,000 irq/sec/队列
  总中断 ≈ 126 队列 × 95,000 ≈ 12M irq/sec
  ```

3) **rx-usecs=64 时**

  ```text
  第一个包到达 → 启动 64 µs 计时器
    ↓ 64 µs 后
  窗口内到达 ~6 个包（64 / 10.5 ≈ 6）
    ↓
  中断频率 = 95,000 / 6 ≈ 15,800 irq/sec/队列
  总中断 ≈ 126 × 15,800 ≈ 2M irq/sec
  ```

4) **实际降幅**
  ```text
  12M → 2M = 约 6 倍
  ```

#### 五、如何查询网卡每秒的软中断次数

查询网卡每秒的软中断次数，通常要看 NET_RX softirq（网络接收软中断） 的增长速率。Linux 没有直接提供“某块网卡每秒 softirq 次数”的单一计数器，因为 softirq 是按 **CPU 统计**的，不按网卡统计

**方法 1：查看 NET_RX softirq 每秒增长（推荐）**
  查看当前 softirq：
  ```text
  # cat /proc/softirqs
  ...
                      CPU0       CPU1       CPU2       CPU3
  NET_RX:          12345678   23456789   34567890   45678901
  NET_TX:            123456     234567     345678     456789
  ...
  ```
  其中：
  - NET_RX：接收方向软中断次数
  - NET_TX：发送方向软中断次数

  计算每秒:
  ```bash
  # watch -n 1 "grep NET_RX /proc/softirqs"
  NET_RX: 1000000 2000000
  NET_RX: 1001200 2001800    //1s后
  ```
  则我们计算出:
  ```text
  CPU0: 1200 次/s
  CPU1: 1800 次/s
  总计: 3000 次/s
  ```

**方法 2：按网卡查看 IRQ 次数（更接近网卡）**
  因为 softirq 不绑定网卡，所以如果你想知道 eth20 触发了多少中断，看 /proc/interrupts：
  ```bash
   # ethtool -i eth20
   driver: mlx5_core
   version: 25.10-1.7.1
   firmware-version: 20.39.3004 (MT_0000000224)
   expansion-rom-version: 
   bus-info: 0000:4b:00.0                                //网卡PCIe地址
   supports-statistics: yes
   supports-test: yes
   supports-eeprom-access: no
   supports-register-dump: no
   supports-priv-flags: yes

  # 查询网卡IRQ范围
  # grep "mlx5_comp.*0000:4b:00.0" /proc/interrupts | awk -F: '{print $1}' | sort -n | head -1     // 起始
  # grep "mlx5_comp.*0000:4b:00.0" /proc/interrupts | awk -F: '{print $1}' | sort -n | tail -1     // 结束

  # cat /proc/interrupts                  //查询中断                     
  # cat /proc/interrupts | grep "981:"    //查询网卡IRQ 981在各CPU上产生的中断
  ```

  |列名	    | 当前值	 |                                      含义                                 |
